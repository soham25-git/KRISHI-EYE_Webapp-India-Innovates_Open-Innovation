## Configuration
| Variable | Purpose | Required |
|----------|---------|----------|
| `NODE_ENV` | Distinguishes `production` builds from `deployment` testing suites. Determines CORS domains globally inside NestJS. | Yes |
| `APP_ENV` | Specifically leverages `demo` flag internally to shift backend to SQLite fallback bypassing PostgreSQL credentials constraint. | Optional |
| `DATABASE_URL` | Connects the active database to the Prisma execution layer (`postgresql://...`). | Yes |
| `JWT_SECRET`, `REFRESH_TOKEN_SECRET` | Asymmetric salt variables responsible for rendering short and long-lived auth sessions uniquely secure. | Yes |
| `PORT` | Local network binding port (Backend APIs actively resolve here). | No (Defaults 3000) |
| `NEXT_PUBLIC_API_URL` | Web Frontend target pointing directly to the deployed endpoint `apps/api`. | Yes |
